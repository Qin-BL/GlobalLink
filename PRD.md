# 游戏化英语学习平台 - 产品需求文档(PRD)

## 1. 产品概述

### 1.1 项目背景
游戏化英语学习平台重构项目，旨在为中文用户打造一个科学、有趣、高效的英语学习环境。通过融合SM-2间隔重复算法和游戏化机制，解决传统英语学习枯燥、效果差的痛点。

### 1.2 产品定位
专为中文用户设计的游戏化英语学习平台，通过科学算法和趣味互动提升学习效果和用户粘性。

### 1.3 产品愿景
成为中文用户首选的英语学习平台，让英语学习变得简单、有趣、高效。

### 1.4 核心问题
1. **首页布局丢失**：需要恢复三大核心学习模块展示
2. **课程中心导航拥挤**：二级侧边导航需要重新设计
3. **主题切换功能失效**：白天/黑夜模式无法正常工作
4. **大量404页面**：功能不完整，用户体验差
5. **页面布局问题**：响应式设计需要优化
6. **缺乏系统性内容**：需要完善课程练习体系

### 1.5 核心价值
- **科学性**：基于SM-2算法的智能复习机制
- **趣味性**：游戏化设计提升学习动力
- **系统性**：完整的学习路径和进度追踪
- **个性化**：根据用户水平提供定制化内容

## 2. 用户分析

### 2.1 目标用户群体

#### 主要用户群体
- **英语初学者**（占比40%）
  - 年龄：18-35岁
  - 特征：英语基础薄弱，学习动力强
  - 需求：系统性基础学习，简单易懂的界面

- **英语进阶者**（占比35%）
  - 年龄：20-40岁
  - 特征：有一定英语基础，希望提升词汇量和语法
  - 需求：针对性练习，进度可视化

- **英语爱好者**（占比25%）
  - 年龄：16-50岁
  - 特征：对英语有兴趣，希望保持和提升水平
  - 需求：多样化内容，社交互动功能

### 2.2 用户画像

#### 张小明（初学者代表）
- 年龄：25岁，程序员
- 痛点：工作需要但英语基础差，时间碎片化
- 目标：快速掌握基础词汇和语法
- 使用场景：通勤时间、工作间隙

#### 李小红（进阶者代表）
- 年龄：28岁，市场专员
- 痛点：词汇量不够，语法掌握不扎实
- 目标：提升商务英语水平
- 使用场景：下班后、周末时间

### 2.3 用户需求分析

#### 功能性需求
1. 词汇学习和记忆强化
2. 语法练习和应用
3. 口语发音训练
4. 学习进度追踪
5. 个性化学习计划

#### 非功能性需求
1. 界面简洁美观
2. 操作流畅响应快
3. 内容权威可靠
4. 多设备同步
5. 离线学习支持

## 3. 功能需求

### 3.1 核心功能模块

#### 3.1.1 词汇量训练（Word Blitz）
**功能描述**：快节奏词汇记忆游戏，提升词汇量

**详细需求**：
- 闪卡式词汇展示
- 中英文切换测试
- 难度等级分类（初级/中级/高级）
- 计时挑战模式
- 正确率统计
- SM-2算法智能复习

**交互流程**：
1. 选择难度等级
2. 开始词汇测试
3. 看英文选中文/看中文选英文
4. 实时反馈正误
5. 显示统计结果
6. 推荐复习词汇

#### 3.1.2 语法掌握（Sentence Builder）
**功能描述**：通过句子构建练习掌握英语语法

**详细需求**：
- 词汇拖拽组句功能
- 语法规则提示
- 句型分类练习
- 错误纠正机制
- 语法知识点解释
- 进度追踪

**交互流程**：
1. 选择语法类型
2. 查看乱序词汇
3. 拖拽组成正确句子
4. 系统校验语法
5. 显示正确答案和解释
6. 记录学习进度

#### 3.1.3 口语练习（Chinese-English）
**功能描述**：中英文对照口语练习，提升口语表达

**详细需求**：
- 中文提示，英文表达
- 语音识别评分
- 发音纠正建议
- 常用对话场景
- 录音回放功能
- 发音对比分析

**交互流程**：
1. 选择对话场景
2. 查看中文提示
3. 录制英文表达
4. AI评分和建议
5. 查看标准发音
6. 重复练习改进

#### 3.1.4 课程中心
**功能描述**：系统性英语课程学习平台

**详细需求**：
- 分级课程体系（A1-C2）
- 主题分类（商务/旅游/日常等）
- 学习路径规划
- 课程进度管理
- 学习证书系统
- 个性化推荐

**页面结构**：
- 主导航：课程分级
- 侧边导航：主题分类（重新设计，解决拥挤问题）
- 内容区：课程列表和详情
- 进度条：学习完成度

#### 3.1.5 学习统计和排行榜
**功能描述**：学习数据可视化和社交激励

**详细需求**：
- 个人学习统计
- 学习时长记录
- 掌握词汇数量
- 学习连续天数
- 全球/好友排行榜
- 成就徽章系统

### 3.2 系统功能

#### 3.2.1 用户系统
- 用户注册/登录
- 个人资料管理
- 学习偏好设置
- 数据备份恢复

#### 3.2.2 主题系统
- 白天/黑夜模式切换（修复现有bug）
- 自定义主题颜色
- 字体大小调节
- 界面布局选项

#### 3.2.3 数据同步
- 云端数据存储
- 多设备同步
- 离线数据缓存
- 学习进度备份

- **平板端**：768-1199px，导航收缩，布局调整
- **手机端**：<768px，单列布局，底部导航

## 5. 交互设计

### 5.1 交互原则
- **简单直观**：操作流程清晰，学习曲线平缓
- **即时反馈**：操作结果立即显示
- **进度可见**：学习进度实时更新
- **错误友好**：错误提示明确，提供解决方案

### 5.2 核心交互流程

#### 5.2.1 首次使用流程
1. 欢迎页面 → 快速注册
2. 英语水平测试 → 个性化设置
3. 功能引导 → 开始学习

#### 5.2.2 日常学习流程
1. 首页查看 → 选择学习模块
2. 开始练习 → 完成任务
3. 查看结果 → 获得反馈
4. 复习建议 → 持续改进

#### 5.2.3 主题切换流程
1. 设置入口 → 主题选项
2. 选择模式 → 即时预览
3. 确认应用 → 全局生效

### 5.3 手势和快捷键
- **拖拽**：句子构建中的词汇排序
- **滑动**：词汇卡片切换
- **双击**：查看详细释义
- **长按**：显示操作菜单

## 6. 技术需求

### 6.1 技术架构
- **前端框架**：Next.js 13+ (App Router)
- **UI框架**：React 18+
- **样式方案**：Tailwind CSS + CSS Modules
- **状态管理**：Zustand
- **数据库**：SQLite (开发) / PostgreSQL (生产)
- **ORM**：Prisma

### 6.2 核心技术特性
- **服务端渲染**：提升首屏加载速度
- **静态生成**：优化SEO和性能
- **增量静态生成**：动态内容更新
- **API路由**：后端逻辑处理

### 6.3 性能要求
- **首屏加载**：< 2秒
- **页面切换**：< 500ms
- **离线支持**：核心功能可离线使用
- **缓存策略**：智能缓存用户数据

### 6.4 兼容性要求
- **浏览器**：Chrome 80+, Firefox 78+, Safari 14+, Edge 80+
- **移动端**：iOS 14+, Android 8+
- **分辨率**：320px - 2560px

### 6.5 安全要求
- **数据加密**：用户敏感信息加密存储
- **API安全**：JWT token认证
- **HTTPS**：全站HTTPS加密
- **XSS防护**：输入验证和转义
